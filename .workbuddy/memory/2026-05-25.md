# 2026-05-25 工作记录

## 集装箱玉米模块月度趋势图修复

### 问题
用户反馈：
1. 图表必须选到2024年才能画出图形，但这样累计销售量就不对了
2. 画出的图跟参考图有出入

### 根因
- `monthlyData`（月度趋势图数据）和 `deptData`（累计文字+表格）共用同一套日期过滤逻辑
- 当 `startDate` 设为2026年时，2024/2025年数据被过滤掉，图表只剩一条线
- 缺失月份用 `0` 填充，导致折线掉到底部，而参考图是断线

### 修复
- `calcContainerData` 中拆分为两个独立逻辑：
  - `deptData`：仍按日期范围过滤（用于累计文字和表格）
  - `monthlyData`：不限制日期范围，只限制品种/经营部/包装方式/纳入签约，并限制为近3年数据
- `renderContainerChart` 中缺失值从 `0` 改为 `null`，并设置 `spanGaps: false` 实现断线效果

### 修改文件
- `app.js`：`calcContainerData()` 日期过滤逻辑拆分 + `renderContainerChart()` 缺失值处理

## 工作簿复用与文件名 clarity 修复

### 问题
用户反馈：
1. 各 tab 需要的文件名称不一致（运营台账 vs 运营台账贴表），容易混淆
2. 前面已经导入的表格，切换到其他 tab 时还要再导一次

### 根因
- `getContainerWorkbook()` 错误地回退到 `contractWorkbook`（运营台账贴表.xlsx），但贴表和台账的列结构完全不兼容（贴表用 `26年纳入签约合同`、`26年合同量-签约`，台账用 `纳入签约量统计`、`合同签订量-万吨`）
- `switchTab` 没有动态检测可用 workbook 并更新空状态提示
- UI 文字把 签约/最后-公里 都列为可复用来源，误导用户

### 修复
- **修正 `getContainerWorkbook()`**：只回退到 `lastmileWorkbook`，移除 `contractWorkbook`
- **新增 `getLastmileWorkbook()`**：最后-公里也可复用 `containerWorkbook`，实现双向共享
- **`calcLastmileData()`**：改用 `getLastmileWorkbook()` 代替直接访问 `lastmileWorkbook`
- **`switchTab` 动态 UI**：新增 `updateTabWorkbookStatus()`，切换 tab 时自动检测可用 workbook，空状态从"请上传..."变为"✅ 数据已就绪"
- **HTML 文字修正**：
  - Container 上传区小字："（或与签约/最后-公里共享）" → "（或与最后-公里共享）"
  - Container 空状态小字："支持复用签约/最后-公里模块..." → "支持与最后-公里模块共享..."
  - Container 复用提示：增加 ".xlsx" 后缀明确文件类型
- **各 tab 统一处理**：intent/contract/settle/lastmile/container 全部调用 `updateTabWorkbookStatus()`

### 数据源映射（修正后）
| 模块 | 所需文件 | 可复用来源 |
|-----|---------|-----------|
| 意向销售 | 1-意向统计.xlsx | 无 |
| 销售签约 | 运营台账贴表.xlsx | 无 |
| 销售结算 | 销售发货清单.xlsx | 无 |
| 最后-公里 | 运营台账.xlsx | 集装箱玉米 |
| 集装箱玉米 | 运营台账.xlsx | 最后-公里 |

### 修改文件
- `app.js`：`getContainerWorkbook()` + `getLastmileWorkbook()` + `calcLastmileData()` + `switchTab()` + `updateTabWorkbookStatus()`
- `index.html`：container tab 文字修正

## 文件名统一为「销售运营台账」+ 复用状态增强

### 问题
1. 用户反馈：最后一公里已上传，但切换到集装箱玉米仍显示「请上传」，复用识别不生效
2. 所有界面文字中的「运营台账」需统一加上「销售」前缀

### 修复
- **全量替换「运营台账」→「销售运营台账」**：
  - `index.html`：lastmile/container 的 card-title、upload zone small、空状态文字、复用提示
  - `index.html`：contract 的 card-title、small、空状态文字 → 「销售运营台账贴表」
  - `app.js`：`switchTab()` 中 `updateTabWorkbookStatus()` 的文件名参数
- **增强复用状态显示**：
  - `updateTabWorkbookStatus()` 新增 `reuseFrom` 参数，复用时显示「（复用自XX）」
  - `switchTab()` 中 lastmile/container 分支显式计算 `isReused`，并传递来源名称
  - Container 的 `reuseHint` 显示逻辑改为基于 `isReused`
- **修复 `processLastmileFile()`**：不再替换整个 `innerHTML`（会导致 fileInput 丢失、拖拽失效），改为只替换 `.upload-placeholder` 的内容

### 修改文件
- `app.js`：`switchTab()` + `updateTabWorkbookStatus()` + `processLastmileFile()` + 注释
- `index.html`：全量「运营台账」→「销售运营台账」
